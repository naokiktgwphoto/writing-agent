# 選手特集（インタビュー）パイプライン

## インプット

- インタビュー文字起こし（または音声）
- 選手プロフィール（年齢・所属・ポジション・経歴）
- 媒体名・想定文字数・連載か単発か

## 媒体別フォーマット

### Number Web（松澤寛政・吉川大紀など）

**タイトル形式**
```
「引用キーフレーズ」解説テキスト
```
- 例：「なんとかなるよ」レイダースとドラフト外契約、"日本人初"の手前で松澤寛政が見ているもの
- キーフレーズは記事の核心となる選手の言葉から取る。

**章立て**
- `##` で章を区切る。
- 章タイトルは情景・転機・心理状態を示唆する体言止め。
  - 例：「ひとりの春」「指名なき夜」「契約のあと　再び仲間の中で」
- `◇　◇` で時制転換や場面切替を示す。

**文体・構造**
- 冒頭：現在地のシーン → `◇　◇` で時制を遡る、という構成が典型。
- 著者が観察者として地の文に入る。
  - 例：「その落差を『ちょっと』と言えることが、この男の強さだ」
  - 例：「話術ではなく、思考の構造そのものなのだ」
- 締め：余韻・暗示で終わる。状況を断言せず、その先を示唆する。
  - 例：「それはもう手が届く距離かもしれない」

### 4years.（朝日新聞）

対象：学生アスリート・卒業直後の社会人1年目（廣田悠人、安藤謙太、東田隆太郎など）

**タイトル形式**

パターンA（テーマ説明型）：
```
選手名、テーマ・状況　「選手の言葉（引用）」
```
- 例：「京大WR廣田悠人、けがを越え取り戻した感覚「京大史上一番のWRに」」
- 例：「東田隆太郎、関学から富士通へ「今はまだ10のうち4くらい」新人DBが見据えるもの」

パターンB（属性＋名前型）：
```
テーマの説明　選手名・所属、キャリアの特徴
```
- 例：「立命館QB安藤謙太　関学一家に生まれ、ライバル校を選んだ2年生が本格デビュー」

タイトル末尾に選手の「言い切る言葉」を引用で入れることが多い。

**構成**
1. リード（1段落）：選手プロフィール + 今季/今試合の活躍 + テーマ設定 + キーフレーズ（選手の言葉）
2. 具体的なプレーシーンまたは直近の試合エピソードから入る。
3. 経歴・背景（出身・他競技からの転向・家族・きっかけなど）。
4. 成長過程・現状の課題（コーチや周辺人物のコメントを交える）。
5. 今季の位置づけ・チーム状況。
6. 締め：選手の目標・決意の言葉 + 著者の一文で包む。

**引用源**
- 選手本人（メイン）
- コーチ・監督（能力評価や指導方針）：小見出しを「コーチ名「コメントのキーフレーズ」」として独立させることもある
- 家族・先輩・同期（選手の人間性・背景の補強）
- 複数の引用源を組み合わせて立体的に描く。コーチや関係者の長めの引用も厭わない。

**文体**
- 章立て（`##`）は使わず、小見出しで区切る（小見出しは説明的・テーマ要約型）。
- 引用は長めでも可。話し言葉のトーンをそのまま生かす。
- スタッツや試合経過を具体的に記述（ヤード数・クオーター・プレーの詳細）。
- 著者の観察・解釈を要所に差し込む（「〜と苦笑する」「〜が、その言葉の裏に〜があった」）。
- 写真が複数入る前提で、写真に対応する地の文を置く。

**その他の媒体（Sportiva・週プレ・BBMなど）**
- 同一題材は切り口・エピソードを各媒体で重複させない。
- 各媒体のトーンに合わせて調整（週プレはよりエンタメ寄り、BBMは競技情報寄りなど）。

## 構造の組み立て方（共通）

### 1. 構造軸（テーマ）を一本決める
記事全体を貫く「問い」や「葛藤」を最初に設定する。
- 例：「1.5軍の壁」「経験ゼロからの逆算」「日本を離れる選択」

### 2. 冒頭の入り方
- 具体的なシーン・情景・感覚から入る（抽象的な紹介文から始めない）。
- 例：「飛行機で泣いたのは覚えている」「パスポートを握りしめていた」

### 3. 本文の展開

#### 連載形式（2〜3回）
- 各回に「転機」を一つ置く。
- 時系列で進めながら、核心のエピソードでドラマを作る。
- 中見出しは「出来事＋示唆」の形式。
  - 例：「王子スタジアム、決勝点を蹴った夜」「残り1席だった飛行機」

#### 引用の使い方
- 選手の言葉を短く引用し、その後に著者の解釈を入れる。
- 謙遜・逡巡と確信が同居する瞬間を書く（「謙遜さを挟みながらもそう言い切る」のような観察）。
- 引用は話し言葉のトーンを保つ。整えすぎない。

### 4. 媒体ごとの出し分け
同一取材で複数媒体に書く場合、切り口・エピソードを重複させない。
媒体ごとに使うエピソードを事前に割り振る。

## 禁止事項

- 「——」（em-dash）を使わない。
- 記事の冒頭でプロフィール紹介から入らない（シーンから始める）。
- 締めに「今後の活躍が期待される」などのAI的常套句を使わない。
- 曖昧な意味づけで段落を閉じない（具体的な事実・引用・観察で閉じる）。
