# 解説・考察記事パイプライン

試合後の分析、選手比較、展望など、インタビューではなく著者の考察を主軸にした記事。
主な媒体：Sportiva

## インプット

- 試合の映像・観戦メモ
- 過去の対戦データ・スタッツ
- 分析の「問い」（例：「なぜ勝てたのか」「誰が次の脅威か」）

## タイトル形式

```
事実の要約。分析的な問い？将来展望も示唆
```

- 試合結果の事実から入り、「？」で分析の問いを立てる。
- 例：「井上尚弥がネリを粉砕。無敵のモンスターにパンチを当てるには？中谷潤人との将来的な対戦も期待」
- 過剰な煽りはしないが、「？」や分析的フックで読者を引き込む。

## 構成

1. **リード（2段落）**：試合の概要（事実）+ この記事で論じることの提示。
2. **本文（`##` 章立て）**：
   - 第1章：試合の振り返り（ハイライト・勝負の分岐点）
   - 第2章：技術・戦術分析（著者の仮説）
   - 第3章：今後の展望・対戦候補
3. 写真は `◆` でキャプション的に説明を添える。
   - 例：`◆2R、2分過ぎ。ドネアの左が炸裂し、右瞼をカットした直後`
   - 写真は分析の裏付けとして使う。「このパンチが〜の例」のように本文の仮説と対応させる。

**リードの書き方：**
- 1段落目：試合の経緯・結果の事実（スコア・ダウン数・舞台の特殊性など）
- 2段落目：「この試合で改めて見えた〜を振り返り、今後の〜を考える」のように記事の目的を提示
- コメント引用は原則なし（選手・HC発言ではなく著者の分析が主）

**技術・戦術分析の書き方：**
- 具体的なラウンド数・秒数・パンチの軌道など数値で裏付ける。
- 「〜という特質に共通性が見えてくる」のように仮説を立てる。
- 自ら反論・留保を加える：「ただ、これはあくまでも"過去"の〜。当然アップデートがあるだろう」。

## 文体

- 著者の仮説・分析を前面に出す。コーチ・選手のコメントは不要。
- 「〜だろう」「〜かもしれない」など推測を含む（断言しすぎない）。
- 読者が競技の基礎知識を持つ前提で書く（初心者向けの補足説明は不要）。
- 章タイトルは説明的・分析的（体言止めより文章型）。
  - 例：「Sバンタム級戦線は役不足。モンスターには近距離のフックが当たりやすい？」
- 仮説に対して自ら反論・留保を加える（「あくまでも"過去"のモンスター。当然アップデートがあるだろう」）。

## 禁止事項

- 「——」（em-dash）を使わない。
- 根拠なく断言しない（「〜に違いない」より「〜だろう」）。
- 曖昧な常套句を使わない（「さらなる成長が期待される」など）。
