# 試合レポート パイプライン

## インプット

- 実況音声記録（文字起こし）
- 公式ドライブチャート（Time Start/End、得点）
- 公式スタッツ
- HCコメント・選手コメント

## アウトプット構成（この順番で書く）

### 1. タイトル
```
【アメフト】◯◯、△△に◇-◇で勝利　メイントピック
```
- フォーマット厳守。過剰な煽りは避ける。

### 2. リード（1段落）
- 試合概要（スコア・会場・日付）
- MVP的な活躍をした選手
- 両チームのシーズン文脈（それまでの戦績）
- この結果が持つ意義

### 3. 本文（時系列＋小見出し）
おおむね以下の流れ：
1. Q1: 開幕ドライブ・試合の入り
2. Q2: 流れの変化
3. Q3: 突き放す/拮抗
4. Q4: 応戦・クロージング
5. スタッツ解説
6. HCコメント・選手コメント

### 4. 結び
- 次戦情報＋HCのフレーミングコメント
- 最終行：試合全体を総括する一文

## 時間表記ルール

- 「第◯Q残り◯分◯秒」の残り時間形式で統一。
- 公式ドライブチャートのTime Start/Endと一致させる。
- 経過時間形式（例：「2Q 2:02」）は使わない。
- 本文中に括弧で時間を補足しない（地の文で残り時間形式を使う）。

## 禁止事項

- `(kick)` などのプレー注記を本文に入れない。
- `(残り◯分)` などの括弧補足を入れない。
- 得点に結びつかないドライブは詳述しない（省略）。
- 細かいプレーシーケンスの列挙をしない。
- 「アウェー」→「アウェー戦」と書く。

## 出力前チェックリスト

- [ ] 「——」を使っていないか
- [ ] タイトルは指定フォーマットか
- [ ] リードに「概要・MVP・両者の戦績文脈・意義」が揃っているか
- [ ] 外国人選手＝初出フルネーム／以降姓、日本人＝姓のみになっているか
- [ ] 時間は「第◯Q残り◯分◯秒」形式でドライブチャートと一致しているか
- [ ] 本文に `(kick)` や括弧補足が残っていないか
- [ ] 詳述したドライブはすべて得点に結びついているか
- [ ] 「アウェー戦」表記になっているか
- [ ] 曖昧な常套句が混入していないか
- [ ] 結びに次戦情報＋HCコメント、最終行に総括が入っているか
